Running corinet

1. Running corinet from the command line

Usage

At the command prompt, type:
corinet -?|-t
corinet [-v|-q] fileName

Flags

  • -? produces a help message
  • -t test known compatibility issues; currently it tests the 32-bit random number generator
  • -v check fileName for errors only; do not run the experiment or generate code
  • -q run in quiet mode, without progress indicator

Argument fileName

An xml file in corinet markup language format. Either:
  • a file in the current working directory, not starting with '.', '\' or '/'
    corinet Exp1.xml
  • a file relative to CORINETROOT, starting with '.' or '..'
    corinet ./work/test/Exp1.xml
  • a file specified by a fully qualified path
    corinet C:\corinet\work\test\Exp1.xml (on Windows)
    corinet /home/corinet/work/test/Exp1.xml (on Linux)

Program output

  • A progress indicator shows the progress of the experiment. This can be supressed by using flag '-q' to run CORINET in quiet mode. In that case, only error messages will be displayed.
  • Results are written in either plain text or xml file format. The type of file, where it is written and what is being written is specified in the experiment setup file.

Troubleshooting

  • $CORINETROOT/bin should be in the search path.
  • When the xml input file contains an error, an error message detailing the type and location of the error is printed out and the program is aborted.
  • If corinet does not produce a detailed error message because of an unforeseen error, or crashes during execution, please send the offending.xml file to info [at] corinet [dot] org.

2. Running corinet from Matlab

Usage

At the Matlab command prompt or in an .m file use:
corinet fileName;
corinet(fileName,flag);
fileName is a file in the current working directory only.

Help

For more help on the available Matlab functions, see the files in $CORINETROOT/code/matlab.

Troubleshooting

  • $CORINETROOT/code/matlab should be in the Matlab path.
  • The progress indicator writes one line per cycle to the Matlab command prompt. This can be surpressed by using flag '-q' to run corinet in quiet mode.

3. Running corinet from R

Usage

At the R command prompt or in an .r file use:
corinet fileName;
fileName is a file in the local directory only. Flags are not allowed.

Help

For more help on the available R functions, see the files in $CORINETROOT/code/r.

Troubleshooting

  • Source $CORINETROOT/code/r/sourceCorinet.R at the start of your session.
  • The progress indicator does not write to the R prompt until the end of the experiment.
