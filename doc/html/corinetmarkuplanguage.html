corinet Markup Language

The corinet markup language specifies several file formats:

Files of type <experimentList> contain 1 or more references to experiment files. Each of these experiment files is executed in the order of their appearance in the list file.

Files of type <experiment> contain the definitions of networks, tasks, and descriptions of how networks and tasks are combined in an execution.

Files of type <languageDefinition> serve two purposes: firstly, they define additional constraints on the validity of experiment files by defining admissable types and parameters for specific network and task elements. Secondly, files of this type can serve to generate C++ classes for new network and task elements.

The main languageDefinition file (languageDefinition.xml) contains all currently supported network and tasks elements. It provides additional information on how to write experiment setup files. Other languageDefinition files are local to the system and form, together with the C++ code generated from them, sandboxes for testing of new network and task elements.

Files of type <results> contain the results of experiments.
