Writing <experiment> Files

Files containing the setup information for experiments are written in an xml file format specified by the corinet markup language. Familiarity with xml concepts, such as elements, tags, attributes, schemas etc. is required.

Currently, information for writing setup files comes from three sources:

  • the documentation generated from the experimentSetup.xsd schema file
  • the official languageDefinition.xml file defines and documents further constraints on admissible types of network and task elements.
  • the experiments from published papers in $CORINETROOT/examples/Experiments.These examples are grouped by paper and are well documented. Earlier papers contain simple experiments and are thus simulated with simple setup files. Later papers contain more complex experiments and use more complex features from the corinet markup language.
